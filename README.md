# AI Novel Studio: DeepSeek & Cline 執筆自動化ツール

このツールは、VS Code 拡張機能の **Cline** と **DeepSeek-V4-Flash API** を活用して、詳細な心理描写を持つ短編小説を自動生成するための執筆支援システムです。

## ⚠️ 免責事項・利用規約（必ずお読みください）

1.  **個人利用限定**: 本ツールはあくまで個人が娯楽目的でAIとの創作を楽しむためのものです。
2.  **公開・収益化の禁止**: 本ツールを用いて生成したコンテンツを、小説投稿サイト、SNS、その他グローバルなプラットフォームへ公開すること、および有料販売等の収益化に利用することは固く禁止します。
3.  **自己責任**: 本ツールの利用（API料金の発生、生成内容、実行結果等）に関して、開発者は一切の責任を負いません。すべて利用者の自己責任において使用してください。

## ✨ 本ツールの機能

| 機能 | 説明 |
|------|------|
| **新規作品作成** (`create`) | タイトルを入力すると `novels/#番号 タイトル/` フォルダと `settings.md` / `manuscript.md` のテンプレートを自動生成します。 |
| **執筆開始** (`start`) | 作品一覧から対象を選ぶと、Cline が `settings.md` を読み込んで DeepSeek-V4-Flash で本文を生成し、`manuscript.md` に保存します。 |
| **バージョン管理** (`versions`) | `manuscript.md` を再生成・上書きする前に、旧版を `versions/manuscript_v{N}.md` として自動バックアップします。`versions` コマンドで履歴一覧を確認できます。 |
| **グローバルルール** (`.clinerules`) | 執筆言語・文字数目標・カスタムコマンドなど、Cline が常時参照するプロジェクト共通の設定ファイルです。 |
| **設定ドリブン執筆** (`settings.md`) | 作品ごとのキャラ・世界観・タグ・参考URL・擬音設定をもとに、一貫性のある本文を生成します。 |
| **キャラクター口調制御** (`settings.md`) | `## キャラクター口調` セクションで各キャラの話し方・語尾・口癖を指定すると、セリフに個性が出ます。 |
| **擬音ルール指定** (`settings.md`) | `## 擬音・効果音ルール` セクションで使用する擬音と避ける表現を指定し、文体に合った擬音描写を実現します。 |
| **ふりがな自動付与** (`settings.md`) | 文化庁『常用漢字表』を基準に、表外漢字・難読漢字・固有名詞の初出時に `漢字(ふりがな)` 形式で自動付与します。 |
| **参考文章によるスタイル学習** (`reference.md`) | 作品フォルダの `reference.md` に好きな文章を貼り付けると、`start` 実行時に文体・テンポ・語彙として学習させられます。 |
| **Windows UTF-8 対応** | ストリーミング出力時の日本語・記号の文字化けを自動補正します。 |

## 🛠 セットアップ手順

### 1. 必要なソフトウェアのインストール
- **VS Code (Visual Studio Code)**: [公式サイト](https://code.visualstudio.com/)からインストール。
- **Python**: [公式サイト](https://www.python.org/)から 3.10 以上をインストール。
- **重要**: インストール画面で「Add Python to PATH」に必ずチェックを入れてください。

### 2. VS Code 拡張機能の導入
VS Code の拡張機能メニュー（Ctrl+Shift+X）から以下をインストールします。
- **Cline** (旧 Claude Dev)
- **Python** (Microsoft公式)

### 3. プロジェクト環境の構築（仮想環境）
プロジェクトフォルダを VS Code で開き、ターミナル（Ctrl+J）で以下のコマンドを順番に実行して、独立した実行環境（仮想環境）を作成・有効化します。

**Windowsの場合:**
```bash
# 仮想環境の作成
python -m venv .venv
# 仮想環境の有効化
.venv\Scripts\activate
```

**Mac/Linuxの場合:**
```bash
# 仮想環境の作成
python3 -m venv .venv
# 仮想環境の有効化
source .venv/bin/activate
```
※ターミナルの先頭に (.venv) と表示されれば成功です。

### 4. ライブラリのインストール
仮想環境が有効な状態（先頭に (.venv) がある状態）で、必要なライブラリを一括インストールします。

```bash
pip install openai python-dotenv
```

### 5. APIキーとセキュリティ設定

DeepSeek API は従量課金制です。先にクレジットをチャージし、その残高から使用した分だけ差し引かれます。

- 初回は最小額の `$2` チャージで始めて問題ありません。
- 2万文字以内の短編小説を試す用途であれば、最初のTOKEN課金額は最小の `$2` で十分です。
- ただし、生成回数を増やしたり、何度も全文を書き直したりすると消費は増えます。

1. DeepSeek の公式サイトとAPI管理画面を開きます。
  - 公式サイト: https://www.deepseek.com/
  - API管理画面: https://platform.deepseek.com/
2. DeepSeek Platform にログインし、API Keys 画面から新しいAPIキーを作成します。
3. APIキー名は用途が分かるものにしてください。例: `novel-api`
4. 発行直後に表示されるAPIキーを控えてください。再表示できない場合があります。
5. プロジェクトのルートフォルダに .env という名前のファイルを作成し、以下を書き込みます。

```plaintext
DEEPSEEK_API_KEY=あなたのAPIキー(sk-...)
```

6. `.env` は機密情報なので、外部公開や共有をしないでください。

### 6. Cline の初期設定
1. VS Code の左サイドバーから Cline を開きます。
2. 設定（歯車アイコン）をクリックし、以下を入力します。
  - API Provider: DeepSeek
  - API Key: あなたのAPIキー
  - Model: deepseek-chat (最新のFlashモデルが適用されます)
3. プロジェクトのルートにある .clinerules を読み込ませるため、一度チャットで「ルールを確認して」と指示してください。

## 🚀 使い方

### コマンド操作（Clineのチャット欄で入力）

- **`create`**:
  - 新規プロジェクト作成。
  - ターミナルでタイトルを入力すると、`novels/#番号 タイトル` フォルダと以下のファイルが生成されます。
    - `settings.md` — キャラ・世界観・擬音ルール・口調などの設定ファイル
    - `reference.md` — 参考文章を貼り付けるスタイル学習用ファイル
    - `manuscript.md` — 本文出力先
- **`start`**:
  - 執筆開始。
  - 作品を選択すると、Clineが自動的に `settings.md` を読み込んで執筆を開始します。
- **`versions`**:
  - バージョン履歴の確認。
  - 作品を選択すると、`versions/` フォルダ内に保存された旧バージョンの一覧（保存日時・ファイルサイズ）を表示します。
  - 再生成や修正のたびに旧 `manuscript.md` が `manuscript_v1.md`, `manuscript_v2.md` … と自動保存されます。

#### バージョン管理の仕組み

再生成を実行すると、上書き前に以下の場所へ自動バックアップされます。

```
novels/#1 タイトル/
  manuscript.md          ← 常に最新版
  settings.md
  versions/
    manuscript_v1.md     ← 1回目の生成内容
    manuscript_v2.md     ← 2回目の生成内容（修正後など）
    …
```

旧バージョンは削除されません。いつでも `versions/` フォルダを直接開いて過去の原稿を参照できます。

### reference.md の使い方

`create` 実行時に各作品フォルダへ自動生成される `reference.md` は、文体・スタイルを学習させるための専用ファイルです。

#### 貼り付け方

1. 参考にしたい作品（小説サイトなど）を開く
2. 気に入った文章・シーンを選択してコピーする
3. `novels/#番号 タイトル/reference.md` を開き、コメント行の下に貼り付けて保存する
4. `start` を実行すると自動的に読み込まれ、文体学習に使われる

#### ポイント

- **pixiv などログインが必要なサイトは自動取得できません**。このファイルへの手動貼り付けが有効です。
- 複数の作品から抜粋して貼り付けても構いません。
- `settings.md` の `参考URL` にアクセスできるURLを書いた場合は自動取得を試みます。取得できなかった場合は `reference.md` を参照します。
- 貼り付ける文章が多いほど学習精度が上がりますが、`settings.md` を肥大化させずに済みます。

### settings.md の書き方

`create` 実行後に生成される `settings.md` は、小説の設計図です。ここが具体的であるほど、出力される本文の精度が上がります。

#### 1. 基本情報を書く

以下の項目は作品全体の方向性を決めます。

- `タイトル`: 作品名をそのまま記入します。
- `形式`: 短編、連作短編、一章完結などを書きます。
- `管理番号`: 自動採番されるので通常は変更不要です。
- `タグ`: `#獣人 #学園モノ` のように、作品の属性をスペース区切りで並べます。
- `擬音・擬態語の使用`: `true` または `false` を記入します。
- `ふりがな`: `true` または `false` を記入します。`true` にすると、文化庁『常用漢字表』基準で難読漢字・固有名詞に自動で `漢字(ふりがな)` 形式のふりがなが付きます。
- `参考URL(小説サイトなど)`: 文体や雰囲気の参考にしたい作品URLを書きます。不要なら空欄で構いません。

#### 2. 登場人物を書く

登場人物は、名前だけでなく「立場」「種族」「性格」「関係性」まで書くと安定します。

記入のコツ:

- 主人公と相手役は必ず分けて書く
- 年齢や学年があるなら入れる
- 口調や性格の特徴を短く添える
- 関係性の緊張感や距離感を書く

#### 3. キャラクター口調を書く

`## キャラクター口調` セクションに各キャラクターのセリフの特徴を書くと、AIがキャラごとに個性的な会話を生成します。

例:

```md
## キャラクター口調
- 颯太: 素直になれない。短文が多く、ぶっきらぼう。でも語尾がたまに柔らかくなる。
- まこと: 落ち着いた語り口。相手を観察してから話す。「…そうか」「まあいい」をよく使う。
```

#### 4. 擬音・効果音ルールを書く

`擬音・擬態語の使用: true` の場合、`## 擬音・効果音ルール` セクションが参照されます。積極的に使う擬音と避けたい表現を明記すると、文体に合ったトーンで生成されます。

例:

```md
## 擬音・効果音ルール
- 使用する擬音の例: とくん、ふわっ、ぎゅっ、ドキドキ
- 避ける表現: ズドーン、ドカーン などの過剰な激しさ
- 密度: 少なめ
```

#### 5. 物語の着地点（結末）を書く

最後にどうなってほしいかを先に決めておくと、AIが途中の展開を組み立てやすくなります。

例:

- 初キスで両想いになる
- まだ付き合わないが、お互いの好意を自覚して終わる
- 切ない余韻を残しつつ再会の約束で終わる

#### 6. 世界観・シチュエーションを書く

舞台、季節、文化、匂い、空気感、社会的なルールなどをここに書きます。本文の雰囲気に直結するので、遠慮なく細かく書いてください。

例:

```md
## 世界観・シチュエーション
- 舞台: 現代的な獣人共生社会の私立高校
- 季節: 秋。少し冷たい風が吹く昼休み
- 恋愛観: 保守的な空気がある
- 匂い描写: `登場人物X`は森や焚き火のような落ち着く匂いがする
```

#### 7. 執筆指示を書く

文体、描写の厚み、心理描写の強さ、参考URLの扱い方など、本文生成時の細かい指示を書きます。

記入のコツ:

- 「何を強く描くか」を書く
- 「避けたいこと」があれば書く
- 参考URLがある場合は、どの要素を参考にするか明記する
- `擬音・擬態語の使用` が `true` の場合は勢い重視、`false` の場合は心理描写重視などの方針を書く

例:

```md
## 執筆指示
- 参考URLがある場合、その作品の文体やテンポ感を分析して反映すること
- 擬音・擬態語の使用が false の場合は、心理描写と情景描写を厚めにすること
- 初対面の緊張感と、距離が縮まる瞬間の甘さを丁寧に描くこと
```

#### 8. settings.md を詳しく書く順番

迷ったら、次の順番で埋めると書きやすいです。

1. `基本情報` で作品のジャンルと条件を決める
2. `登場人物` で主役同士の関係を固める
3. `キャラクター口調` でセリフの個性を設定する
4. `物語の着地点（結末）` で終わり方を先に決める
5. `世界観・シチュエーション` で舞台と空気感を足す
6. `執筆指示` で文体や演出の希望を補足する

#### 9. よくある設定の書き方

- タグを増やしたい場合: `#甘々 #切ない #年上攻め` のように追加します。
- 参考作品を使いたい場合:
  - **小説家になろう・カクヨムなど認証不要のサイト** → `参考URL(小説サイトなど)` にURLを書くと自動取得されます。
  - **pixiv など要ログインのサイト** → URLを書いても取得できません。`reference.md` に文章を直接貼り付けてください。
- 擬音を使いたい場合: `擬音・擬態語の使用: true`
- 擬音を抑えたい場合: `擬音・擬態語の使用: false`
- ふりがなを付けたい場合: `ふりがな: true`
  - 基準: 文化庁『常用漢字表』（平成22年内閣告示・2136字）
  - 表外漢字・難読な読み・固有名詞初出時に `漢字(ふりがな)` 形式で付与されます

`settings.md` の情報量が少ないと、本文も汎用的になりやすいです。短くてもよいので、作品固有の匂い・感情・結末は必ず入れてください。

## 📁 ディレクトリ構造
- `.clinerules`: Cline への命令定義
- `.env`: APIキー保存用（**公開厳禁**）
- `.gitignore`: 機密ファイル等の除外設定
- `scripts/`: 管理スクリプトおよび執筆エンジン
- `novels/`: 生成された小説の保存先
  - `#N タイトル/settings.md`: キャラ・世界観・擬音・口調などの設定
  - `#N タイトル/reference.md`: スタイル学習用の参考文章（手動貼り付け）
  - `#N タイトル/manuscript.md`: 生成された本文（常に最新版）
  - `#N タイトル/versions/`: 上書き前の原稿バックアップ
